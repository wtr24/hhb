Create `backend/models/ohlcv.py`:
```python
from sqlalchemy import Column, String, Numeric, BigInteger, TIMESTAMP, Index
from .base import Base

class OHLCV(Base):
    __tablename__ = "ohlcv"

    time = Column(TIMESTAMP(timezone=True), primary_key=True, nullable=False)
    ticker = Column(String(20), primary_key=True, nullable=False)
    open = Column(Numeric(18, 6))
    high = Column(Numeric(18, 6))
    low = Column(Numeric(18, 6))
    close = Column(Numeric(18, 6))
    volume = Column(BigInteger())
    source = Column(String(20))

    __table_args__ = (
        Index("ix_ohlcv_ticker_time", "ticker", "time"),
    )
```

Create `backend/models/__init__.py`:
```python
from .base import Base
from .ohlcv import OHLCV

__all__ = ["Base", "OHLCV"]
```

Create `backend/alembic.ini` — set `script_location = alembic`, set `sqlalchemy.url` to empty string (overridden by env.py from DATABASE_URL env var).

Create `backend/alembic/env.py` using the sync engine pattern from research (NOT async — Alembic uses sync engine even in async projects):
- Import `Base` from `models.base`
- Import `models.ohlcv` as side-effect import (noqa: F401)
- Set `target_metadata = Base.metadata`
- `run_migrations_online()`: read `DATABASE_URL` from `os.environ`, create sync engine with `create_engine()`, run migrations synchronously
- `run_migrations_offline()`: use url from env var, configure context with `literal_binds=True`
- Strip any `+asyncpg` from DATABASE_URL to ensure psycopg2 driver is used

Create `backend/alembic/versions/0001_initial_hypertables.py`:
- `revision = "0001"`, `down_revision = None`
- `upgrade()`: Create `ohlcv` table with columns matching the OHLCV model, then immediately call `op.execute("SELECT create_hypertable('ohlcv', 'time', if_not_exists => TRUE)")`, then create index `ix_ohlcv_ticker_time`
- `downgrade()`: `op.drop_table("ohlcv")`

Also create empty `backend/alembic/versions/__init__.py`.
