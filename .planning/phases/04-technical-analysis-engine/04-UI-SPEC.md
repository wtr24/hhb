## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline tag padding, badge inner padding |
| sm | 8px | Button padding, panel header inner padding |
| md | 16px | Between control groups, picker section gaps |
| lg | 24px | Modal internal section spacing |
| xl | 32px | Not used in this phase (dense terminal layout) |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Panel headers use `py-0.5` (2px vertical) — matches existing `ChartPanel.tsx` header rhythm. Preserved as-is.
- Chart sub-pane heights: main candle pane = 75% of chart container height; each oscillator sub-pane = 25% of chart container height. These are proportional height splits, not spacing tokens.
- Touch targets for `[Indicators ▾]`, `[Fib]`, and `[EW]` buttons: minimum 24px height (keyboard-first terminal, no mobile requirement per REQUIREMENTS.md out-of-scope section).

| Label | 10px | 400 (regular) | 1.2 | Chart axis labels, badge text, sub-pane labels — matches existing `chartConfig.ts` fontSize: 10 |

- 14px is explicitly not used — the existing terminal renders at 12px body and 10px chart labels. Phase 4 matches this.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0a0a0a` `--color-terminal-bg` | Chart backgrounds, modal/picker background, sub-pane backgrounds, drawing tool overlay background |
| Secondary (30%) | `#1a1a1a` `--color-terminal-border` | Panel borders, picker section dividers, oscillator pane separators, button borders, Fibonacci level lines (dimmed at 0.3 opacity) |
| Accent (10%) | `#ff9900` `--color-terminal-amber` | Reserved for specific elements listed below |
| Positive signal | `#00d084` `--color-terminal-green` | Candle up bars, bullish pattern badges, positive win-rate badges |
| Negative/Destructive | `#ff4444` `--color-terminal-red` | Candle down bars, bearish pattern badges, p-value fail badges |
| Dim | `#404040` `--color-terminal-dim` | Inactive/disabled state text, placeholder text, "insufficient data" badge text |

Accent (`#ff9900`) is reserved for:
1. Active indicator names in the indicator picker (selected state)
2. `[Indicators ▾]`, `[Fib]`, and `[EW]` button text (normal state)
3. `[Fib]` and `[EW]` button border when drawing mode is active (highlight active tool)
4. Candlestick pattern badge text: `Hammer: 65% win | n=43 | p=0.03`
5. Chart pattern label at breakout point: `H&S 72%` and `[experimental]` tag
6. Elliott Wave number/letter labels placed on the chart (1, 2, 3, 4, 5, A, B, C)
7. Fibonacci level line labels (0.382, 0.618, etc.) — text only, not the line itself
8. Indicator overlay series lines that are "primary" (e.g. EMA 20, RSI line) — secondary lines (e.g. MACD signal line) use `#404040`
9. Pivot point horizontal lines (Standard/Woodie's/Camarilla/Fibonacci/DeMark) — S1/R1 lines in amber, S2/R2 in dim

## Interaction Contracts

### Indicator Picker Modal

- Trigger: `[Indicators ▾]` button in expanded chart panel header
- Opens: inline dropdown anchored below the button, width 280px, max-height 400px with internal scroll
- Structure: 7 collapsible sections (Moving Averages, Momentum, Trend Strength, Volatility, Volume, Market Breadth, Pivot Points). Sections start collapsed except the last-used section.
- Each item: indicator name + checkbox. Checked = active on chart.
- When checked: indicator immediately computed and drawn. Picker stays open to allow multi-select.
- Dismiss: clicking outside the picker, pressing Escape, or clicking `[Indicators ▾]` again.
- Active indicator count badge: small amber number badge on the `[Indicators ▾]` button showing count of active indicators. Hidden when 0.

### Indicator Parameter Editor

- Trigger: clicking the label of an active indicator in the expanded chart (label rendered as text overlay in top-left of chart or sub-pane)
- Opens: small inline popover (not full modal) with numeric inputs for each parameter (e.g. Period: [14], Source: [Close])
- Apply: pressing Enter or clicking away immediately recomputes and redraws the indicator
- Cancel: pressing Escape restores prior values
- No "Save" button — changes take effect on Enter/blur

### Fibonacci Drawing Tool

- Activated by: `[Fib]` button — button border changes to amber to signal active drawing mode
- Drawing mode cursor: default browser cursor (no special cursor; Bloomberg convention)
- Step 1: user clicks a bar on the chart — marks swing point 1 (a crosshair dot rendered in amber appears)
- Step 2: user clicks a second bar — levels are drawn immediately as horizontal lines across full chart width
- Cancel: Escape key or clicking `[Fib]` button again (button border returns to border color)
- Timeframe change: all Fibonacci drawings cleared (D-15)
- Ticker change: all drawings cleared (D-14)
- Multiple Fibonacci drawings allowed: each click pair adds a new set; no limit

### Elliott Wave Labelling Tool

- Activated by: `[EW]` button — button border changes to amber
- Sequential mode: each click on a bar places the next label in the sequence: 1, 2, 3, 4, 5, A, B, C
- Label placement: text tag at bar high (peaks: 1, 3, 5, B) or bar low (troughs: 2, 4, A, C)
- After label 3: Fibonacci validation fires. Badge renders inline: pass (`#00d084`) or fail (`#ff4444`) with rule description
- After label 4: overlap check fires. Same badge pattern.
- After C: sequence complete. `[EW]` button returns to inactive state. New click on `[EW]` starts fresh sequence.
- Cancel: Escape clears all placed labels and exits EW mode
- Ticker/timeframe change: all EW labels cleared (D-14)

### Oscillator Sub-Panes

- Layout: main candle pane = 75% of chart height; each oscillator sub-pane = 25% height
- When 2 oscillators active: each sub-pane is 12.5% of total height (chart container fixed height per D-01 Bloomberg fixed-viewport rule)
- Pane separator: 1px `#1a1a1a` horizontal line
- Sub-pane label: indicator name + period in top-left corner, 10px amber text
- Sub-pane zero line: 1px `#1a1a1a` dashed line at relevant reference value (RSI: 50, MACD: 0)
- Reference lines: RSI 70/30 rendered as 1px `#1a1a1a` lines

### Candlestick Pattern Badges

- Position: small text overlay at the right edge of the pattern bar, vertically offset above the candle high
- Format: `{Pattern Name}: {win}% win | n={count} | p={p_val}` — e.g. `Hammer: 65% win | n=43 | p=0.03`
- Below n=30: `{Pattern Name}: insufficient data`
- Badge background: `#0a0a0a` (transparent in practice — text on chart)
- Font: 10px monospace, `#ff9900` amber

### Chart Pattern Overlays

- Shaded region: `rgba(255, 153, 0, 0.08)` fill spanning the detected pattern bar range
- Label: rendered at the breakout bar — format `{Pattern} {confidence}%` + ` [experimental]` suffix
- All 7 patterns use identical visual treatment — only the label text changes
- Example: `H&S 72% [experimental]`
- Font: 10px monospace, `#ff9900`

### Pivot Point Lines

- Rendered as horizontal full-width lines in expanded chart view
- Method selector: button group in panel header (STANDARD | WOODIE | CAM | FIB | DEMARK) — active method highlighted in amber border
- Line labels rendered at right price scale as text
- PP (pivot): `#ff9900` amber solid line
- R1/S1: `#ff9900` amber dashed line
- R2/S2 and beyond: `#404040` dim dashed line

## Implementation Notes for Executor

1. **No new files outside established paths.** All new backend code in `backend/analysis/`. All new frontend components in `frontend/src/components/equity/`. Follow CONTEXT.md canonical refs exactly.

2. **lightweight-charts v5 pane API.** Use `chart.addSeries(LineSeries, options, paneIndex)` where paneIndex >= 1 for oscillator sub-panes. Use `chart.panes()[n].setHeight(pixels)` to set sub-pane height. Do NOT use v4 `addLineSeries()` — removed in v5. Source: RESEARCH.md Pattern 3.

3. **TA-Lib routing.** TA-Lib is primary for all indicators it covers. pandas-ta is fallback only. Route documented per indicator in RESEARCH.md phase requirements table (TA-01 through TA-08).

4. **Chart container is fixed height.** The Bloomberg layout uses `h-screen flex flex-col` on the outer div (Phase 03 D-01 decision). Do NOT add scrolling to the chart area. Oscillator sub-panes compress the candle pane — this is intentional.

5. **All components use TERMINAL constants from `frontend/src/lib/theme.ts`.** Never hardcode hex values — import `TERMINAL.AMBER`, `TERMINAL.BG`, etc.

6. **Picker modal scrolls internally** — the modal container has a fixed max-height (400px) with `overflow-y: auto` and the scrollbar styled via the existing `::-webkit-scrollbar` rules in `index.css`.

7. **No persistence for drawings.** No DB writes for Fibonacci or EW labels. State lives in React component state only. Session-only per D-14.
