  <action>
Create `docker-compose.dev.yml` (NO `version:` header):

```yaml
services:
  api:
    volumes:
      - ./backend:/app
    command: >
      sh -c "alembic upgrade head &&
             uvicorn api.main:app --host 0.0.0.0 --port 8000 --reload"
    environment:
      - PYTHONDONTWRITEBYTECODE=1

  beat:
    volumes:
      - ./backend:/app

  worker:
    volumes:
      - ./backend:/app

  frontend:
    build:
      context: ./frontend
      target: dev
    ports:
      - "3000:5173"
    volumes:
      - ./frontend:/app
      - /app/node_modules
    command: npm run dev -- --host 0.0.0.0
```

Key details:
- api: adds `--reload` flag to uvicorn, mounts `./backend:/app` for live code changes, sets `PYTHONDONTWRITEBYTECODE=1` to avoid __pycache__ pollution
- beat: mounts `./backend:/app` so task definitions update on save
- worker: mounts `./backend:/app` (note: celery worker does NOT auto-reload — must restart container on task changes, but volume mount means `docker compose restart worker` picks up changes without rebuild)
- frontend: overrides build target to `dev` stage (Vite dev server), remaps port `3000:5173` (Vite port, not Nginx port 80), mounts `./frontend:/app` with anonymous volume `/app/node_modules` to prevent host node_modules overwrite, runs `npm run dev -- --host 0.0.0.0` for Docker accessibility

Usage: `docker compose -f docker-compose.yml -f docker-compose.dev.yml up`
